## Overview

Track the status of any universal transaction on Push Chain using its transaction hash. This function provides real-time progress updates and returns the transaction receipt once confirmed.

**Key features:**
- **Poll transaction status** until confirmation or timeout
- **Progress tracking** with detailed event callbacks
- **Flexible configuration** for polling intervals and timeouts
- **Identical to `wait()`** but can be called independently with any transaction hash

## Track Universal Transaction

**_`pushChainClient.universal.trackTransaction(txHash, {options}): Promise<TransactionReceipt>`_**

```typescript
const receipt = await pushChainClient.universal.trackTransaction(
  '0xe2302bd21ab0902f37cb605d491ce5f95ee35ce4083405dddf3657d782acae35',
  {
    waitForCompletion: true,
    progress: (event) => {
      console.log(`${event.id}: ${event.message}`);
    },
  }
);
```

<PushAPIReference>

| **Arguments**          | **Type** | **Default** | **Description**                                       |
| ---------------------- | -------- | ----------- | ----------------------------------------------------- |
| _`txHash`_             | `string` | -           | The transaction hash to track (must be valid 32-byte hex string with `0x` prefix). |
| `options.waitForCompletion` | `boolean` | `true` | Whether to wait for transaction confirmation before returning. |
| `options.chain`        | `CHAIN`  | `CHAIN.PUSH_TESTNET_DONUT` | The chain where the transaction was submitted. |
| `options.progress`     | `(event: ProgressEvent) => void` | `undefined` | Callback function to receive real-time progress updates during tracking. |

</PushAPIReference>

<Details summary="Advanced Arguments" className="alert alert--minimal code">

| Arguments      | Type                                   | Default | Description                                                                                                                            |
| -------------- | -------------------------------------- | ------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| `options.advanced.pollingIntervalMs` | `number` | `2000` | Interval in milliseconds between status checks (minimum: 500ms). |
| `options.advanced.timeout` | `number` | `60000` | Maximum time in milliseconds to wait for confirmation (60 seconds default). |
| `options.advanced.rpcUrls` | `Partial<Record<CHAIN, string[]>>` | `{}` | Custom RPC URLs for querying transaction status. |

<Details summary="ProgressHook Type and Response" className="alert alert--minimal code">

| Field                | Type                                 | Description                                                                     |
| -------------------- | ------------------------------------ | ------------------------------------------------------------------------------- |
| `progress`           | `Object`                             | The progress of the transaction tracking.                                      |
| `progress.id`        | `string`                             | Unique identifier for the progress event.                                       |
| `progress.title`     | `string`                             | Brief title of the progress event.                                              |
| `progress.message`   | `string`                             | Detailed message describing the event.                                          |
| `progress.level`     | `INFO` \| `SUCCESS` \| `WARNING` \| `ERROR` | Severity level of the event.                                                    |
| `progress.timestamp` | `string`                             | ISO-8601 timestamp when the event occurred (e.g. `2025-06-26T15:04:05.000Z`). |

| ID                | Title                               | Message                                                                                   | Level   |
| ----------------- | ----------------------------------- | ----------------------------------------------------------------------------------------- | ------- |
| `TRACK-TX-01`     | Tracking Started                    | Starting to track transaction: `<tx hash>`                                                | INFO    |
| `TRACK-TX-02`     | Querying Status                     | Polling transaction status (attempt `<n>`)                                                | INFO    |
| `TRACK-TX-03`     | Status Update                       | Transaction status: `<pending/confirmed/failed>`                                          | INFO    |
| `TRACK-TX-99-01`  | Tracking Complete                   | Transaction confirmed in block `<block number>`                                           | SUCCESS |
| `TRACK-TX-99-02`  | Tracking Failed                     | `<error message>` (timeout or transaction failure)                                        | ERROR   |

</Details>

## Using `tx.wait()` vs `trackTransaction()`

Both methods return identical transaction receipts, but serve different use cases:

### `tx.wait()` - Immediate Tracking

Use when you just sent a transaction and want to wait for confirmation:

```typescript
const tx = await pushChainClient.universal.sendTransaction({
  to: '0x35B84d6848D16415177c64D64504663b998A6ab4',
  value: BigInt(1000),
});

// Wait for confirmation immediately
const receipt = await tx.wait();
```

### `trackTransaction()` - Independent Tracking

Use when you need to track a transaction by hash (e.g., from storage, user input, or different session):

```typescript
// Track any transaction hash
const receipt = await pushChainClient.universal.trackTransaction(
  '0xe2302bd21ab0902f37cb605d491ce5f95ee35ce4083405dddf3657d782acae35',
  { waitForCompletion: true }
);
```

## Progress Hook Integration

The `trackTransaction()` function supports progress hooks at two levels:

### Per-Transaction Progress Hook

Register a progress callback specific to this tracking operation:

```typescript
const receipt = await pushChainClient.universal.trackTransaction(
  txHash,
  {
    progress: (event) => {
      console.log(`[TRACK] ${event.id}: ${event.message}`);
    },
  }
);
```

### Orchestrator-Level Progress Hook

Set a global progress hook during client initialization to receive events from all operations:

```typescript
const pushChainClient = await PushChain.initialize(universalSigner, {
  network: PushChain.CONSTANTS.PUSH_NETWORK.TESTNET,
  progressHook: (event) => {
    console.log(`[GLOBAL] ${event.id}: ${event.message}`);
  },
});
```

### Using `tx.progressHook()` Method

For transactions returned by `sendTransaction()`, you can register a progress hook that will receive tracking events during `wait()`:

```typescript
const tx = await pushChainClient.universal.sendTransaction({
  to: '0x35B84d6848D16415177c64D64504663b998A6ab4',
  value: BigInt(1000),
});

// Register progress hook for this transaction
tx.progressHook((event) => {
  console.log(`[TX] ${event.id}: ${event.message}`);
});

// Wait will trigger tracking events
await tx.wait();
```

## Error Handling

Handle common tracking errors gracefully:

```typescript
try {
  const receipt = await pushChainClient.universal.trackTransaction(
    txHash,
    {
      waitForCompletion: true,
      advanced: {
        timeout: 30000, // 30 seconds
      },
    }
  );
  
  if (receipt.status === 0) {
    console.error('Transaction failed on-chain');
  } else {
    console.log('Transaction successful!');
  }
} catch (error) {
  if (error.message.includes('Timeout')) {
    console.error('Transaction tracking timed out');
  } else if (error.message.includes('Invalid')) {
    console.error('Invalid transaction hash format');
  } else {
    console.error('Tracking error:', error.message);
  }
}
```

### Common Error Scenarios

| Error Type | Cause | Solution |
|------------|-------|----------|
| **Invalid hash format** | Transaction hash is not a valid 32-byte hex string | Ensure hash starts with `0x` and is 66 characters long |
| **Timeout** | Transaction not confirmed within timeout period | Increase `advanced.timeout` or check network congestion |
| **Transaction not found** | Hash doesn't exist on the specified chain | Verify the transaction hash and chain parameter |
| **Network error** | RPC connection issues | Check RPC URLs in `advanced.rpcUrls` or network connectivity |

## Tracking Multiple Transactions

You can track the same transaction multiple times or track multiple different transactions:

```typescript
// Track same transaction multiple times (e.g., from different components)
const receipt1 = await pushChainClient.universal.trackTransaction(txHash, {
  waitForCompletion: true,
});

const receipt2 = await pushChainClient.universal.trackTransaction(txHash, {
  waitForCompletion: true,
});

// Both return identical receipts
console.log(receipt1.hash === receipt2.hash); // true

// Track multiple transactions concurrently
const receipts = await Promise.all([
  pushChainClient.universal.trackTransaction(hash1, { waitForCompletion: true }),
  pushChainClient.universal.trackTransaction(hash2, { waitForCompletion: true }),
  pushChainClient.universal.trackTransaction(hash3, { waitForCompletion: true }),
]);
```
