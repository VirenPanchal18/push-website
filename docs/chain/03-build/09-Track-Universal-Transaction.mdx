## Overview

Track the status of any universal transaction on Push Chain using its transaction hash. The function polls until the transaction is confirmed and returns the full receipt — or streams real-time progress events to your callback.

You can use `trackTransaction()` **independently** from `sendTransaction()` — pass any hash you have stored, received from another session, or want to re-poll.

> **Note:** `trackTransaction()` and `tx.wait()` return identical receipts. See [wait() vs trackTransaction()](#using-txwait-vs-tracktransaction) for when to use each.

## Track Universal Transaction

**_`pushChainClient.universal.trackTransaction(txHash, {options}): Promise<UniversalTxResponse>`_**

```typescript
const response = await pushChainClient.universal.trackTransaction(
  '0x6dcea443f7262ec22f40180402dc12cf39437f647859d4539abfe2fdfc6eb03f',
  {
    waitForCompletion: true,
    progressHook: (event) => {
      console.log(`${event.id}: ${event.message}`);
    },
  }
);
```

<PushAPIReference>

| **Arguments** | **Type** | **Default** | **Description** |
| ------------- | -------- | ----------- | --------------- |
| _`txHash`_ | `string` | - | Transaction hash to track. Must be a valid 32-byte hex string prefixed with `0x`. |
| `options.waitForCompletion` | `boolean` | `true` | When `true`, waits for on-chain confirmation before resolving. When `false`, returns immediately after the first status check. |
| `options.chain` | `CHAIN` | `CHAIN.PUSH_TESTNET_DONUT` | The chain on which the transaction was submitted. |
| `options.progressHook` | `(event: ProgressEvent) => void` | `undefined` | Callback invoked at each tracking step with a `TRACK-TX-*` progress event. |

</PushAPIReference>

<Details summary="Advanced Arguments" className="alert alert--minimal code">

| Arguments | Type | Default | Description |
| --------- | ---- | ------- | ----------- |
| `options.advanced.pollingIntervalMs` | `number` | `2000` | Milliseconds between polling attempts. Minimum: `500`. |
| `options.advanced.timeout` | `number` | `60000` | Maximum milliseconds to wait before throwing a timeout error. |
| `options.advanced.rpcUrls` | `Partial<Record<CHAIN, string[]>>` | `{}` | Custom RPC URLs to use when querying status. |

<Details summary="ProgressHook Type and Response" className="alert alert--minimal code">

| Field | Type | Description |
| ----- | ---- | ----------- |
| `progress.id` | `string` | Unique identifier for the progress event. |
| `progress.title` | `string` | Short title of the step. |
| `progress.message` | `string` | Detailed human-readable description. |
| `progress.level` | `INFO` \| `SUCCESS` \| `WARNING` \| `ERROR` | Severity of the event. |
| `progress.timestamp` | `string` | ISO-8601 timestamp (e.g. `2025-06-26T15:04:05.000Z`). |

| ID | Title | Message | Level |
| -- | ----- | ------- | ----- |
The inline `progressHook` on `trackTransaction()` receives **`TRACK-TX-*`** events:

| ID | Title | Message | Level |
| -- | ----- | ------- | ----- |
| `TRACK-TX-01` | Tracking Started | Starting to track transaction: `<tx hash>` | INFO |
| `TRACK-TX-02` | Querying Status | Polling transaction status (attempt `<n>`) | INFO |
| `TRACK-TX-03` | Status Update | Transaction status: `<pending/confirmed/failed>` | INFO |
| `TRACK-TX-99-01` | Tracking Complete | Transaction confirmed in block `<block number>` | SUCCESS |
| `TRACK-TX-99-02` | Tracking Failed | `<error message>` (timeout or on-chain failure) | ERROR |

The `response.progressHook(cb)` callback (registered after the call) replays buffered **`SEND-TX-*`** events.

</Details>

</Details>

### Route 2 — Track a CEA Execution on an External Chain

When you send a Route 2 transaction (execution on an external chain via CEA), `sendTransaction()` returns a hash from the origin. Track it the same way — the SDK resolves the CEA-side confirmation automatically.

{`
// customPropHighlightRegexStart=universal\.trackTransaction
// customPropHighlightRegexEnd=\\);
// customPropGTagEvent=track_transaction_route2

import { PushChain } from '@pushchain/core';
import { ethers } from 'ethers';
import \* as readline from 'node:readline/promises';

const rl = readline.createInterface({ input: process.stdin, output: process.stdout });

async function main() {
  console.log('🔍 Route 2: Send from Sepolia → Execute on Sepolia via CEA, then track');

  // 1) Set up wallet on Ethereum Sepolia
  const wallet = ethers.Wallet.createRandom();
  console.log('📝 Wallet:', wallet.address);

  const provider = new ethers.JsonRpcProvider('https://ethereum-sepolia-rpc.publicnode.com');
  const signer = wallet.connect(provider);

  // 2) Convert to Universal Signer and initialize Push Chain client
  const universalSigner = await PushChain.utils.signer.toUniversal(signer);
  const pushChainClient = await PushChain.initialize(universalSigner, {
    network: PushChain.CONSTANTS.PUSH_NETWORK.TESTNET,
  });

  await rl.question(':::prompt:::Send Sepolia ETH to: ' + wallet.address + ' to continue.');

  // 3) Route 2: target an external chain contract via CEA
  console.log('📤 Sending Route 2 transaction...');
  const txResponse = await pushChainClient.universal.sendTransaction({
    to: {
      address: '0x0000000000000000000000000000000000042101',
      chain: PushChain.CONSTANTS.CHAIN.ETHEREUM_SEPOLIA,
    },
    value: PushChain.utils.helpers.parseUnits('0.001', 18),
  });

  console.log('Transaction hash:', txResponse.hash);

  // 4) Track the transaction — SDK handles CEA confirmation resolution
  console.log('🔍 Tracking Route 2 transaction...');
  const response = await pushChainClient.universal.trackTransaction(
    txResponse.hash,
    {
      waitForCompletion: true,
      progressHook: (event) => {
        console.log(\`[\${event.level}] \${event.id}: \${event.message}\`);
      },
      advanced: {
        timeout: 120000, // 2 min — external chains need more time
        pollingIntervalMs: 3000,
      },
    }
  );

  console.log('✅ Confirmed at block:', response.blockNumber.toString());
  console.log('Status:', response.status === 1 ? 'SUCCESS' : 'FAILED');
}

## Using `tx.wait()` vs `trackTransaction()`

Both methods return identical `TransactionReceipt` objects. The difference is how you call them:

### `tx.wait()` — Chain off `sendTransaction()`

Use when you just sent a transaction in the same session and want to wait inline:

```typescript
const tx = await pushChainClient.universal.sendTransaction({
  to: '0x35B84d6848D16415177c64D64504663b998A6ab4',
  value: BigInt(1000),
});

// Inline wait — no separate hash lookup needed
const receipt = await tx.wait();
console.log('Confirmed at block:', receipt.blockNumber.toString());
```

### `trackTransaction()` — Track any hash independently

Use when you have a hash from somewhere else — stored in a database, received from a user, or from a previous session:

```typescript
// Works with any valid Push Chain tx hash
const response = await pushChainClient.universal.trackTransaction(
  '0xe2302bd21ab0902f37cb605d491ce5f95ee35ce4083405dddf3657d782acae35',
  {
    waitForCompletion: true,
    progressHook: (event) => console.log(`${event.id}: ${event.message}`),
  }
);
```

## Error Handling

```typescript
try {
  const response = await pushChainClient.universal.trackTransaction(txHash, {
    waitForCompletion: true,
    advanced: { timeout: 30000 },
  });

  if (response.status === 0) {
    console.error('Transaction reverted on-chain');
  } else {
    console.log('Transaction confirmed:', response.hash);
  }
} catch (error) {
  if (error.message.includes('Timeout')) {
    console.error('Timed out — try increasing advanced.timeout');
  } else if (error.message.includes('Invalid')) {
    console.error('Invalid hash — must be 0x-prefixed 32-byte hex');
  } else {
    console.error('Tracking error:', error.message);
  }
}
```

| Error | Cause | Fix |
|-------|-------|-----|
| **Invalid hash format** | Not a valid 32-byte hex with `0x` prefix | Hash must be 66 characters starting with `0x` |
| **Timeout** | Transaction not confirmed within `advanced.timeout` | Increase timeout or check network congestion |
| **Transaction not found** | Hash doesn't exist on the specified chain | Verify hash and `options.chain` value |
| **Network error** | RPC unreachable | Supply custom URLs via `advanced.rpcUrls` |
