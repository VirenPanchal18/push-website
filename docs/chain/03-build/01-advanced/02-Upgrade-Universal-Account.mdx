## Overview

Upgrades the UEA (Universal Executor Account) to the latest required implementation version. This is a gasless, signature-based operation — no gas fees are charged on the origin chain.

You only need to call this when `getAccountStatus()` reports `uea.requiresUpgrade === true`. Sending universal transactions on an outdated UEA will fail.

> **Note**: `upgradeAccount()` is a no-op if the UEA is already at or above the minimum required version.

## Upgrade Universal Account

**_`pushChainClient.upgradeAccount({options?}): Promise<void>`_**

```typescript
await pushChainClient.upgradeAccount({
  progressHook: (progress) => {
    console.log(`${progress.id}: ${progress.message}`);
  },
});
```

<PushAPIReference>

| **Arguments** | **Type** | **Default** | **Description** |
| ------------- | -------- | ----------- | --------------- |
| `options.progressHook` | `(event: ProgressEvent) => void` | `undefined` | Callback invoked at each upgrade step showing progress. |

</PushAPIReference>

<Details summary="Progress Hook: Type and Response" className="alert alert--fn-args">

Each `ProgressEvent` emitted during the upgrade has the following shape:

```typescript
{
  id: string;        // Step identifier, e.g. "UEA-MIG-01"
  title: string;     // Short human-readable step title
  message: string;   // Description of what is happening
  timestamp: number; // Unix timestamp (ms)
}
```

Progress events fired in sequence:

| **ID** | **Title** | **When** |
| ------ | --------- | -------- |
| `UEA-MIG-01` | Checking status | Fetching current UEA version and upgrade requirement |
| `UEA-MIG-02` | Awaiting signature | Prompting the user to sign the migration payload (EIP-712) |
| `UEA-MIG-03` | Broadcasting | Submitting the migration transaction to Push Chain |
| `UEA-MIG-9901` | Upgrade complete | UEA successfully upgraded — includes new version string |
| `UEA-MIG-9902` | Upgrade failed | An error occurred during the upgrade flow |
| `UEA-MIG-9903` | No upgrade needed | UEA is already at the required version; operation skipped |

</Details>

## Recommended Usage Pattern

Always check `getAccountStatus()` before calling `upgradeAccount()` to avoid unnecessary prompts:

```typescript
const status = await pushChainClient.getAccountStatus();

if (status.uea.requiresUpgrade) {
  await pushChainClient.upgradeAccount({
    progressHook: (progress) => {
      console.log(`${progress.id}: ${progress.message}`);
    },
  });
}
```

      const status = await pushChainClient.getAccountStatus();
      console.log('Account status:', JSON.stringify(status));

        console.log('ℹ️ UEA not yet deployed — no upgrade needed');
        return;
      }

      if (!status.uea.requiresUpgrade) {
        console.log(\`✅ UEA already up to date (version \${status.uea.version})\`);
        return;
      }

      await pushChainClient.upgradeAccount({
        progressHook: (progress) => {
          console.log(\`[\${progress.id}] \${progress.title}: \${progress.message}\`);
        },
      });

      const updated = await pushChainClient.getAccountStatus({ forceRefresh: true });
      console.log(\`✅ Upgrade complete. New version: \${updated?.uea?.version}\`);
    }

    await main().catch(console.error);
